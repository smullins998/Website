Identifying Artificailly Generated Music: SongSpy

Intro
With the recent surge of Artificial Intelligence methods entering popular culture, many individuals and industries are assessing how this revolution will impact them. While it’s anticipated that AI will influence every sector, the audio and entertainment industry is likely to experience significant disruption.
The following model uses Machine Learning to identify artificailly generated music.
Data Used
We gather audio files from popular artists who are being cloned (Drake, The Weekend, ect...) using PyTube. The dataset we gather is 1,000 audio files from various artists:
The feature extraction process is somewhat experimental, but the two features I found to perform classification the best are:

  • MFCCs — Mel Frequency Cepstral Coefficients

  • Spectral Contrast

Both features emulate the audio file's spectrogram representation. MFCCs are heavily used in speech recognition and genre classification.
Understand the Data
We conduct a PCA analysis to better understand the dimensionality of our features, and if there are variances between artists that we can we confident in.
We also build a correlation matrix to understand highly correlated features, and where we may be able to be more specific with feature selection.
As we are analyzing audio files, we can assume that the features exhibit a strong temporal nature and are highly correlated with adjacent features. We can extract every-other feature to get rid of multicolinearity in the dataset, but this ends up halving our feature set resulting in a worse testing accuracy.
Model Selection
We attempt several ML models (Support Vector Machine, Random Forest, KNN, and Multi-Layer Perceptron) to understand which performs the best. MLP was consistently above all others with an average train/test accuracy of ~80%.
The structure of the MLP is simple with one hidden layer of 100 nodes, a RELU activation function, and a softmax output layer for multiclass output. The input values are scaled using a standard scaler, and the model trains over 50 epochs.
Choose your model:
We can see that the MLP classifier is more confident in it's predictions whereas the other models make more low-probability predictions. We can then look to understand where the model was inaccurately classifying files.
Results & Implimentation
We can see the model had trouble with Drake and The Weekend as these styles are somewhat similar. An improvement on top of this would be to train acapellas only as to get more accurate vocal representations. Alternatively, we can add more data to the artists which the model had trouble with.
Now that we have a fairly accurate model prediction, we need to impliment logic to identify if an audio file is artificially generated. The ShazamAPI is used to identify songs and compare it to the model prediction.
  • 1) If the model prediction matches the Shazam prediction, this is a real song,
  • 2) If the Shazam prediction is null or is different from the model prediction, then this song is artifical.

Reverse Momentum

Intro
In this writeup we are going to explore a hypothesis I have stemming from stock trading sentiment around companies and adverse events.

  • Ho:

    Markets are efficient, and Equities reflect the correct market value at any given time, specifically off the back of major events.

  • Ha:

    Equities tend to be oversold on the back of a major event (poor earnings, negative news, economic factors).
The COVID pandemic has stirred sizable retail trading interest. People are more connected and active about how frequently they trade, leading to adverse events being oversold as people rush to get out or get into a trade. We are going to backtest this hypothesis over different time periods. But first, we need to get data.
Data Used

          today_close_price = history['Close'][-1]

          percent_change = ((today_close_price - previous_close_price) / previous_close_price) * 100

          ticker_ret.append(percent_change)
          ticker_loop.append(i)
      except:
          print('error with', i)

  daily_ret_df = pd.DataFrame({'Ticker': ticker_loop, 'pctchange': ticker_ret})
  final_list = []

  daily_ret_df = daily_ret_df[daily_ret_df.pctchange < -10]

  #Logic to filter stocks
  for j in daily_ret_df.Ticker:
      
      mkt_cap = yf.Ticker(j).info['marketCap']
      epoch = yf.Ticker(i).info['firstTradeDateEpochUtc']
      td = datetime.today() - datetime.fromtimestamp(epoch)
      
      if mkt_cap > 5e+9 and td.days > 400:
          final_list.append(j)
      
  return final_list
